#### #pragma once用法总结

防止重复引用造成二义性（主要是为了避免重复编译）,通常有两种方式:

- #ifndef

  ```
  #ifndef _CODE_BLOCK
  #define _CODE_BLOCK
  // code
  #endif// _CODE_BLOCK 
  ```

- #pragma once

  ```
  #pragma once
  ```

- 区别

  #ifndef方式受C/C++语言标准的支持，不受编译器的任何限制；
  #ifndef可以针对一个文件中的部分代码，而#pragma once只能针对整个文件（拷贝的多个相同文件不行）。
  相对而言，#ifndef更加灵活，兼容性好，#pragma once操作简单，效率高。
